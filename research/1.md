# Open-Source AI Agents and Tooling for Portfolio Analysis and Management

## Overview

This report surveys notable open-source AI agents and adjacent tooling that can help an individual investor run periodic portfolio analyses, track a short decision ledger, and explore portfolio allocation ideas without relying on paid SaaS tools. It focuses on projects that can be self-hosted locally, integrate with spreadsheets or CSVs, and leverage large language models (LLMs) or agent frameworks for financial analysis.

## Your Context and Requirements

The user maintains a manually curated portfolio spreadsheet, invests primarily with a long-term fundamental bias, and prefers to run tools locally rather than as cloud SaaS. The user already has strong technical skills and has built a Go + SQLite portfolio tracker with XIRR, suggesting comfort with self-hosting, Docker, and wiring APIs or CLIs together.[1]

Key requirements inferred:

- Free and open-source (no recurring subscription fees for the core tool, though LLM/API costs may exist).
- Local-first or self-hostable, ideally with Docker or desktop binaries.
- Ability to ingest a portfolio file (CSV/Excel or via an internal datastore) and compute risk/return metrics.
- Room to embed personal investing philosophy and maintain a short narrative ledger of each review.
- Preference for general-purpose AI agent frameworks over opaque black-box robo-advisors.

## Shortlist of Best-Fit Projects

### 1. Privatefolio – "The AI Wealth Manager" (tooling + AI hooks)

**What it is**

Privatefolio is an open-source, local-first portfolio, accounting, and personal finance manager marketed as "The AI Wealth Manager" and designed to serve as an "operating system" for personal finances. It supports investment portfolios, accounting, tax planning, and AI workflows and is licensed under AGPL-3.0.[2][3]

**Architecture and logic**

- Built with TypeScript/React, Bun/Node, and SQLite, with desktop apps and Docker images for self-hosting.[3]
- Manages transactions and positions in a local database; provides time-travel over trades and balance history, letting users inspect historical net worth and P&L at any past point.[4]
- Treats itself as a core platform where extension modules can add CSV import formats, new price data providers, advanced analytics, or AI features such as deep research agents or voice interfaces.[3]
- Designed explicitly to integrate with LLMs/AI agents: documentation describes using portfolio and transaction data as inputs to AI workflows for insights and planning.[2][3]

**Pros**

- Local-first with strong privacy: the primary data store is under user control, and the project emphasizes self-hosting (desktop apps or Docker).[3]
- Broad financial scope (stocks, crypto, real estate, expenses, cash flow) so a single platform can hold the full household balance sheet.[3]
- Time-travel feature is effectively a built-in performance ledger; you can inspect how prior allocations performed when reviewing decisions.[4]
- Actively developed, with v2 and Time Travel highlighted in self-hosted and DeFi communities.[5][4]
- Designed to be combined with AI agents rather than being a closed robo-advisor; extensions can implement AI-based analytics tailored to user philosophy.[2][3]

**Cons**

- While the core is free/open-source, there is a premium cloud plan with higher AI limits and faster infrastructure; AI-heavy use may incur LLM/API costs.[3]
- Out of the box, AI functionality is more of an integration pattern than a fully opinionated investing agent; building a custom agent that reflects a personal philosophy still requires work.[2][3]
- UI and import flows are oriented around general-purpose finance rather than India-specific brokers, so some CSV mapping or manual configuration will be needed.

**Fit for your needs**

Privatefolio is an excellent base to hold and visualize the portfolio and to wire into a local AI agent that runs weekly analyses over the internal SQLite or CSV exports. Its time-travel and P&L analytics make it ideal for keeping a short review ledger and checking whether past thesis notes actually played out.[2][3]

### 2. Investment-Portfolio-AI-Agent – ReAct-based portfolio analysis agent

**What it is**

Investment-Portfolio-AI-Agent is a Python + Streamlit project that implements an "Investment Portfolio Risk Assessment AI Agent" using a simple ReAct (Reason + Act) agent flow for portfolio risk analysis, diversification assessment, and basic expected return estimation.[6]

**Architecture and logic**

- Uses the ReAct pattern where the agent goes through explicit Thought → Action → Observation cycles to decide which tools (data fetch, risk calculator, etc.) to call.
- Key features include stock risk profiling (volatility, market cap, return potential), sector allocation and diversification analysis, and expected return calculations based on historical stock performance.[6]
- Offers a Streamlit web interface for uploading a portfolio and interacting with the agent via a browser.
- Built as a single-agent workflow: the LLM reasons about the portfolio and calls deterministic Python functions for metrics.

**Pros**

- Codebase is relatively small and readable; easy to inspect or customize the agent's prompts and tool logic for a personal philosophy.
- Clear separation between LLM reasoning (ReAct loop) and quantitative functions (risk metrics, return calculations), allowing independent verification of math.
- Streamlit frontend is simple to extend for additional views or inputs (e.g., adding a "notes" field to store thesis and weekly ledger comments).
- Focused on portfolio-level insights rather than execution, reducing operational risk.

**Cons**

- Tightly focused on risk/diversification and basic expected return; no multi-asset allocation optimizer or regime detection logic baked in.
- Requires an LLM API key (e.g., OpenAI or similar) to run the agent, which introduces variable API costs; the project itself is free but not costless to run.
- No built-in connection to brokers; portfolios are likely imported via CSV, which is fine for your spreadsheet but requires a data-prep step.

**Fit for your needs**

This project is a strong candidate for a weekly analysis assistant: the ReAct loop is transparent, the risk/diversification outputs align with fundamental investing horizons, and you can layer your own scoring rules or guardrails into the agent prompts and Python tools. It does not manage a full financial life like Privatefolio but is easier to treat as a focused "portfolio doctor".[6]

### 3. FinRobot – Multi-agent platform for financial AI

**What it is**

FinRobot is an open-source AI agent platform designed specifically for financial analysis, built by the AI4Finance Foundation as a successor or complement to FinGPT. It aims to provide a full-stack framework for financial AI tasks such as equity research, forecasting, document analysis, and trading.[7]

**Architecture and logic**

- Implements a layered architecture: a financial AI agents layer (perception, brain, action) on top of financial LLM algorithms and data adapters for market data, company filings, and news.[7]
- Uses multi-agent workflows managed by a Director agent, which orchestrates Assistant, LLM Analyst, and domain-specific Financial Analyst agents.
- Provides a FinRobot Pro equity research assistant that can be locally deployed as a web app, fetching financial data, running multi-agent analysis, and generating research reports.[7]
- Integrates external data sources via API keys (e.g., financial data providers, LLMs) configured through a config file; deployment is via virtualenv or shell script.[7]

**Pros**

- Designed as a platform, allowing custom agent composition and workflows, including portfolio-related analysis; aligns well with an advanced DIY setup.[7]
- Multi-agent architecture encourages separation of concerns (data retrieval vs. analysis vs. narrative), which is useful for debugging and for encoding a philosophy in specific agents.
- Locally deployable web interface with full open-source code and documentation, enabling self-hosting on a personal machine or VPS.[7]

**Cons**

- Complexity is significantly higher than focused portfolio agents; there is a learning curve to understand the architecture and configure new workflows.
- The default equity research use case is more about single-stock or industry analysis than portfolio-level allocation or rebalancing.[7]
- Requires API keys for data and LLMs, so running costs depend on usage volume.[7]

**Fit for your needs**

FinRobot is best seen as a long-term platform if you want to build your own multi-agent investment research and portfolio-assessment system, rather than an out-of-the-box solution for weekly reviews. It is powerful but heavy; combining it with a simpler portfolio data layer (like your Go tool or Privatefolio) could eventually yield a sophisticated, philosophy-aware assistant.[7]

### 4. Claude-Investor and related multi-agent skills

**What it is**

Claude-Investor is an open-source investment analysis agent originally released as GPT-Investor and later adapted to use Claude 3 models; it automates equity research for companies in a given industry. Related projects like Super Hedge Fund Skill turn Claude into a multi-agent hedge-fund-style analyst that runs parallel agents (fundamental, technical, valuation, sentiment) and produces structured reports.[8][9]

**Architecture and logic**

- Claude-Investor workflow: generate a list of tickers for a specified industry, retrieve historical price data, financial statements, news, and analyst ratings, perform sentiment analysis, and rank companies on investment attractiveness.[8]
- Uses LLMs for data orchestration, narrative analysis, and recommendation; quantitative work relies on Python notebooks with deployed data-fetching and analysis routines.
- Super Hedge Fund Skill defines eight specialized agents (fundamental, technical, valuation, sentiment, and several investor-persona agents) and uses rule-based calculations for financial metrics plus LLM summarization to produce markdown reports with risk-adjusted position sizing guidance.[9]

**Pros**

- Very strong at single-stock and industry-level research, which is valuable for feeding your own thesis notes before adding or trimming positions.[8]
- Multi-agent designs (as in Super Hedge Fund Skill) separate deterministic metrics from qualitative reasoning, improving transparency and alignment with fundamental investing.[9]
- Open-source code with notebooks or skill definitions makes it possible to inspect prompt templates, scoring rules, and position-sizing logic.[9]

**Cons**

- Not portfolio-centric: by default, these tools analyze candidate stocks, not the overall allocation, risk profile, or rebalancing strategy.
- Depend on commercial LLM APIs (Claude, OpenAI), so some cost is inevitable even though the repos are free.[9]
- No direct integration with your broker portfolio file; you would need to script your own connection from CSV holdings into the agent queries.

**Fit for your needs**

Claude-Investor and similar skills are best treated as research engines for new ideas and for validating or challenging your fundamental theses, which you then feed into a portfolio-focused tool such as Privatefolio or Investment-Portfolio-AI-Agent. They are ideal to generate the "rationale" lines in your weekly ledger but do not themselves manage portfolio state.[8]

### 5. AI-Hedge-Fund-Crypto and Condor – Crypto-focused multi-agent trading

**What they are**

AI-Hedge-Fund-Crypto is a next-generation algorithmic trading framework for cryptocurrencies that combines a directed acyclic graph (DAG) workflow with technical-analysis strategies and an LLM for portfolio management decisions. Condor is an open-source harness from the Hummingbot ecosystem that orchestrates autonomous trading agents, connecting LLM reasoning to deterministic Hummingbot execution for multi-exchange, multi-strategy trading.[10][11][12]

**Architecture and logic**

- AI-Hedge-Fund-Crypto uses a LangGraph-based computation graph in which separate nodes handle multi-timeframe technical analysis, with signals from different strategies (trend following, mean reversion, momentum, volatility, statistical arbitrage) aggregated via adaptive weighting. An LLM layer considers these signals and risk parameters to make portfolio-level decisions.[10]
- Condor defines a standardized Observe → Orient → Decide → Act loop for agents, where portfolio state and market data are fetched via Hummingbot APIs, strategies are evaluated within guardrails, and deterministic executors carry out trades and record P&L.[12]

**Pros**

- State-of-the-art architectures for autonomous trading agents, particularly in crypto, with strong separation between LLM reasoning and hard execution guardrails.[10]
- Provide inspiration and concrete patterns for how to record agent decisions in a session journal and use that as context for future decisions—a pattern directly relevant to your idea of a short review ledger.

**Cons**

- Primarily geared toward automated or semi-autonomous trading, often intraday or multi-timeframe crypto strategies; not naturally aligned with long-term, India-equity-focused investing.[10]
- Operational complexity (exchange connections, API keys, risk limits) is high relative to a weekly portfolio checkup.

**Fit for your needs**

These projects are more relevant as design references than as direct tools. Their agent loops (Observe → Decide → Act) and journaling concepts can guide how you design a portfolio-review agent that logs each weekly decision and its outcome over time.[10]

## Recommended Stack for Your Use Case

Given your preference for local hosting, long-term fundamental investing, and a manually maintained portfolio spreadsheet, the best approach is to combine a robust portfolio data platform with a thin, philosophy-aware AI agent layer.

### Base layer: Portfolio data and analytics

- Use Privatefolio as the canonical store of all portfolio and transaction data, taking advantage of its CSV import capability and stock/ETF tracking via Yahoo Finance.[2][3]
- Alternatively, continue to use your Go + SQLite tracker and expose read-only views (e.g., CSV exports or a small HTTP API) that an agent can query each week.[1]

### Agent layer: Weekly analysis and ledger

- Start with the Investment-Portfolio-AI-Agent codebase as a template for the weekly analysis agent, but replace its frontend with a CLI or minimal web interface that reads your specific spreadsheet/DB schema.[6]
- Modify the ReAct tools to:
  - Compute per-holding fundamentals and risk metrics (using your own factors).
  - Evaluate alignment with your philosophy (e.g., valuation bands, quality scores, secular themes) via explicit rule functions.
  - Generate an "AnalysisRecord" object for each holding containing: snapshot metrics, narrative, and action recommendation (hold, add, trim, exit).
- Extend the agent to append each weekly run to a small ledger table (in SQLite or a separate CSV) with fields such as date, holdings summary, key decisions, and rationales; this mirrors the journaling and session-learning patterns seen in Condor and multi-agent crypto frameworks.[10]

### Research engine: Idea generation

- Use Claude-Investor or similar multi-agent research tools off to the side when you want deep dives on specific industries or stocks.[8]
- Feed the resulting summaries into your ledger as "thesis" notes, which the weekly agent can reference through a knowledge base or notes table.

## How to Rank and Choose

Based on open-source maturity, transparency of logic, and alignment with your goals, the following ranking is appropriate:

1. **Privatefolio** – Best foundation for portfolio data, analytics, and integration with AI workflows, with strong self-hosting and open-source licensing.[4][3][2]
2. **Investment-Portfolio-AI-Agent** – Best starting point for a focused portfolio-analysis agent using a transparent ReAct loop over deterministic financial tools.[6]
3. **FinRobot** – Powerful but heavy platform for building custom multi-agent financial workflows; ideal if you later want to generalize your agent into a broader research system.[7]
4. **Claude-Investor / Super Hedge Fund Skill** – Excellent for equity research and thesis generation, best used as a research sidecar rather than the central portfolio tool.[9][8]
5. **AI-Hedge-Fund-Crypto / Condor** – Best-in-class agent architectures and journaling patterns, but mainly applicable as design inspiration for your own agent rather than as daily tools.[12][10]

This stack lets you preserve full control of your data, encode your own investment philosophy directly into deterministic scoring rules, and still benefit from modern open-source AI agent designs.
