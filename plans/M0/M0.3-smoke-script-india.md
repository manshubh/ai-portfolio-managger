# M0.3 — India smoke-test prompt script

> Paste-ready prompts for the India MCP smoke session. Session type: **fresh Cursor or Claude Code window**, project-scoped MCPs loaded from `.mcp.json` / `.cursor/mcp.json`. Total cells: **24** (8 tools × 3 tickers). Captures raw outputs to `temp/mcp-smoke-raw-india.md`.
>
> See [plans/M0/M0.3-mcp-servers.md §Smoke-test execution protocol](./M0.3-mcp-servers.md#smoke-test-execution-protocol) for the end-to-end workflow and [config/mcp-servers.md §6](../../config/mcp-servers.md#6-smoke-test-references) for the context.

## 0. Preflight (user actions before opening the chat window)

1. **`uv` must be installed** so `uvx` is on Cursor's spawn PATH; without it, `finstack` and `nsekit` both error on startup and 21 of the 24 cells become "tool not found". On macOS: `brew install uv`. Verify with `uvx --version`. Restart Cursor after install if it was already running.
2. **Start the `nse-bse-mcp` HTTP server** in a dedicated terminal and leave it running:

   ```bash
   npx -y nse-bse-mcp@0.1.5
   # blocks; http://localhost:3000/mcp listening
   ```

3. **Open a fresh Cursor or Claude Code window** in the repo root (`/Users/mrihal/workspace/ai-portfolio-manager`). Start a new chat with no prior context.
4. **Confirm the three MCPs are loaded.** Run one of:
   - Cursor: Settings → MCP → verify `finstack`, `nsekit`, `nse-bse` all green.
   - Claude Code: ask the assistant "List all MCP tools you can call." It should enumerate tools from all three servers. If only finstack+nsekit show up, the `nse-bse-mcp` server (step 2) is not running. If finstack+nsekit are missing, step 1 (uv) is the cause.
5. **Create the scratch capture file** and paste each response into it verbatim, under the matching cell heading:

   ```bash
   mkdir -p temp && printf '# M0.3 — India smoke raw captures\n\n' > temp/mcp-smoke-raw-india.md
   ```

   `temp/` is gitignored.

## 2. SPEC → actual tool-name mapping

SPEC §18.2 lists conceptual tool names; the MCPs ship with different names and signatures. Use the **"Actual tool" column** in the prompts below. Any deviation here (an MCP release changes a name, or adds a needed arg) gets logged in `research/mcp-gaps.md` with `market: india`.

| SPEC name | MCP | Actual tool | Signature (this matrix uses) |
|---|---|---|---|
| `get_fundamentals` | finstack | `company_profile` | `company_profile(symbol)` |
| `get_income_statement` | finstack | `income_statement` | `income_statement(symbol)` — annual by default |
| `get_balance_sheet` | finstack | `balance_sheet` | `balance_sheet(symbol)` — annual by default |
| `get_ratios` | finstack | `key_ratios` | `key_ratios(symbol)` |
| `get_fii_dii` | finstack | `nse_fii_dii_data` | `nse_fii_dii_data()` — **no args, market-wide**; see note under §3 |
| `nse_equity_quote` | nse-bse | `nse_equity_quote` | `nse_equity_quote(symbol)` — unchanged |
| `equity_history` (~1y) | nsekit | `equity_price_history` | `equity_price_history(symbol, period="1Y")` |
| `corporate_events` (±90d) | nsekit | `corporate_actions` | `corporate_actions(symbol, period="3M")` — closest enum to "last ~90d"; for a true ±90d window use explicit `start_date`/`end_date` around today |

## 3. Capture format

For each cell, paste the assistant's raw response into `temp/mcp-smoke-raw-india.md` under a heading of the form:

```
### C<nn> — <tool> · <ticker>
<raw JSON or text response, first 40 lines if long>
```

`nn` counts 01..24 in the order of §4 below. No interpretation needed at capture time — just the verbatim response. If an MCP errors, paste the error text. If the tool is not found or a parameter is rejected, paste that verbatim too.

> **FII/DII coverage caveat.** `nse_fii_dii_data` is market-wide — it does not accept a ticker. Cells C13/C14/C15 all call the same tool with no args; the response is the same across the three rows. We keep three cells for matrix-shape consistency with SPEC §18.2; the finding "tool is market-wide, not per-ticker" is pre-recorded in `research/mcp-gaps.md` with `severity: low` because the M2 aggregator can broadcast a single market-wide snapshot to every ticker's context.

## 5. After capture

Once all 24 cells are captured in `temp/mcp-smoke-raw-india.md`, return to the primary (agent) session and ask the agent to parse the scratch file into `research/mcp-smoke-test-india.md`. The agent will also append any failing / thin cells to `research/mcp-gaps.md` with `market: india`.

Pass criterion reminder: a non-empty structured response containing at least one expected field for the tool (e.g., `company_profile` → `market_cap` or `sector`; `balance_sheet` for a bank → `total_deposits` or `tier_1_ratio`; `equity_price_history` → an OHLC row). An error payload, `{}`, or "N/A / not supported" is a gap.

This script file is transient and may be deleted after the bd ticket closes — the authoritative record is `research/mcp-smoke-test-india.md`.
