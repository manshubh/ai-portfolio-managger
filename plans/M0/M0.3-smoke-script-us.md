# M0.3 — US smoke-test prompt script

> Paste-ready prompts for the US MCP smoke session. Session type: **fresh Cursor or Claude Code window** (separate from the India session), project-scoped MCPs loaded from `.mcp.json` / `.cursor/mcp.json`. Total cells: **18** (6 tools × 3 tickers). Captures raw outputs to `temp/mcp-smoke-raw-us.md`.
>
> See [plans/M0/M0.3-mcp-servers.md §Smoke-test execution protocol](./M0.3-mcp-servers.md#smoke-test-execution-protocol) for the end-to-end workflow and [config/mcp-servers.md §6](../../config/mcp-servers.md#6-smoke-test-references) for the context.

## 0. Preflight (user actions before opening the chat window)

1. **`uv` must be installed** so `uvx` is on Cursor's spawn PATH; without it, `finstack` errors on startup and all 18 cells become "tool not found". On macOS: `brew install uv`. Verify with `uvx --version`. Restart Cursor after install if it was already running.
2. **`nse-bse-mcp` server is NOT required** for this session — the US matrix only calls finstack tools. You can leave the India server stopped. (Registering nsekit + nse-bse in the config is fine; their tools just go uncalled.)
3. **Open a fresh Cursor or Claude Code window** in the repo root. Separate window from the India session — no context bleed.
4. **Confirm finstack is loaded.** Ask the assistant "List all finstack MCP tools." It should enumerate `company_profile`, `income_statement`, `balance_sheet`, `key_ratios`, `sec_filing`, etc. If none surface, re-check `.mcp.json` / `.cursor/mcp.json` and confirm `uvx --version` works from a shell.
5. **Create the scratch capture file** and paste each response into it verbatim, under the matching cell heading:

   ```bash
   mkdir -p temp && printf '# M0.3 — US smoke raw captures\n\n' > temp/mcp-smoke-raw-us.md
   ```

   `temp/` is gitignored.

## 2. SPEC → actual tool-name mapping

SPEC §18.2 lists conceptual tool names; finstack ships with slightly different names. Use the **"Actual tool" column** in the prompts below. Any deviation logged in `research/mcp-gaps.md` with `market: us`.

| SPEC name | Actual tool | Signature (this matrix uses) |
|---|---|---|
| `get_fundamentals` | `company_profile` | `company_profile(symbol)` — returns market_cap, sector, description |
| `get_income_statement` | `income_statement` | `income_statement(symbol)` — annual by default |
| `get_balance_sheet` | `balance_sheet` | `balance_sheet(symbol)` — annual by default |
| `get_ratios` | `key_ratios` | `key_ratios(symbol)` — valuation + profitability + health |
| `get_sec_filing` (10-K) | `sec_filing` | `sec_filing(symbol, filing_type="10-K", count=5)` — **signature matches SPEC** |
| `get_sec_filing` (10-Q) | `sec_filing` | `sec_filing(symbol, filing_type="10-Q", count=5)` |

> **Good news for M9:** finstack's `sec_filing` signature is `(symbol, filing_type, count)` — exactly what SPEC §18.2 assumes. Risk **R4** ("finstack `get_sec_filing` might not expose the SPEC-assumed signature") in the parent plan is cleared by this mapping; no adapter work for M9.

## 5. After capture

Once all 18 cells are captured in `temp/mcp-smoke-raw-us.md`, return to the primary (agent) session and ask the agent to parse the scratch file into `research/mcp-smoke-test-us.md`. The agent will also append any failing / thin cells to `research/mcp-gaps.md` with `market: us`.
