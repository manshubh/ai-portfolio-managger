## 2. Approach

`my_philosophy.py` carries one table per scheme: `(metric_name, fund_key, yaml_threshold_key, op)`. Cleaner than suffix-stripping; easier to extend.

### 2.2 `pass` vs `pass_with_exception` semantics (Q3)

For each check row:

1. Compute `base_pass = compare(value, base_threshold, op)`. Set `row.pass = base_pass`.
2. If a sector exception modifies this row's threshold (matching `yaml_threshold_key`):
   - Compute `eff_pass = compare(value, eff_threshold, op)`.
   - Always set `row.exception` and `row.effective_threshold`.
   - If `base_pass != eff_pass`: set `row.pass_with_exception = eff_pass` (so `true` = lax exception saved row; `false` = strict exception killed row).
   - If `base_pass == eff_pass`: omit `pass_with_exception` (no behavioral change to highlight).
3. Effective pass: `eff_pass if pass_with_exception in row else base_pass`.

`pass_count` / `fail_count` use the effective pass.

This matches the SPEC ¬ß9.3 INFY row literally (`pass: false, pass_with_exception: true`).

### 2.3 Dealbreaker detection

Three kinds:

| Dealbreaker | Trigger |
|---|---|
| `zero_promoter` | `metrics.fund.promoter_pct == 0` AND not (`sector_exception` has `exempt: true`). Skipped if scheme has no `promoter_min` (it always does for both schemes). |
| `governance_red_flag` | `metrics.governance_red_flag == True` (defaults False). |
| `user_thesis_exit` | `metrics.user_thesis_exit == True` (defaults False). |

Investigation ¬ß3 puts the two booleans in M3.4's schema deliverable, but M3.3 fixtures need them today. Read with `metrics.get(key, False)` so absence is graceful ‚Äî M3.4 just formalizes the SPEC ¬ß18.2 wording.

### 2.4 PF/15 graded ladder

```python
def grade(effective_fail_count, dealbreakers):
    if dealbreakers:
        return 0
    if effective_fail_count == 0:
        return 15
    if effective_fail_count <= 2:
        return 8
    return 0
```

### 2.5 `check-thresholds` output shape

```jsonc
{
  "ticker": "INFY",
  "market": "india",          // from philosophy YAML `market` field
  "scheme": "non_financial",
  "checks": [
    { "metric": "roe", "value": 28.7, "threshold": 12, "op": ">=", "pass": true },
    ...
    { "metric": "promoter", "value": 14.5, "threshold": 40, "op": ">=", "pass": false,
      "exception": "it_mnc", "effective_threshold": 10, "pass_with_exception": true },
    ...
  ],
  "pass_count": 8,
  "fail_count": 1,
  "dealbreakers_triggered": [],
  "philosophy_fit_graded": 8
}
```

For `fcf_positive` (boolean check), the row shape is `{ "metric": "fcf_positive", "value": true, "pass": true }` (no `threshold` / `op` ‚Äî matches SPEC ¬ß9.3 INFY row).

### 2.6 `persona --persona my-philosophy` output shape

Wraps the same PF computation in the ¬ß5.2 persona contract:

```jsonc
{
  "ticker": "INFY",
  "persona": "my-philosophy",
  "sub_scores": { "philosophy_fit": 8 },
  "weighted_score": 8,
  "max_score": 15,
  "signal": "neutral",         // 15 ‚Üí bullish, 8 ‚Üí neutral, 0 ‚Üí bearish
  "confidence": 0.5,           // 1.0 / 0.5 / 1.0 mapped from PF tier
  "details": {
    "philosophy_fit": "1 non-dealbreaker fail (pat_cagr_3y); 0 dealbreakers triggered."
  }
}
```

`my-philosophy` is **exempt from `insufficient_data`** per investigation ¬ß5.2 ‚Äî it consumes only summary ratios from `fund.*` plus the YAML, both guaranteed present by Phase 1+2. If a required `fund.*` key is missing, that's an exit-1 bad-input error, not graceful degradation.

The `--scheme` and `--sector-exception` flags belong to `check-thresholds`. For `persona`, the persona itself has to infer them OR ‚Äî for M3.3 simplicity ‚Äî `persona --persona my-philosophy` accepts `--scheme` and `--sector-exception` as well. M3.11 (`full`) handles passing them through.

### 2.7 YAML loader ‚Äî minimal

Front-matter delimiter is the M0.5 schema: file starts with `---\n`, ends front-matter on the next bare `---\n`, body follows. We slice the bytes between those delimiters and feed to `yaml.safe_load`. No need for python-frontmatter or markdown libs ‚Äî keep deps minimal.
