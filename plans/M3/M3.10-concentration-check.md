### 2.2 `prices.json` shape (investigation §6)

```jsonc
{
  "as_of": "2026-04-17",
  "lookback_days": 252,        // informational — vol uses its own 60-day window
  "tickers": {
    "RELIANCE.NS": {
      "dates":  ["2025-04-18", "2025-04-21", ...],   // trading days, ascending
      "closes": [2840.5, 2851.0, ...]                // native ccy; scale-free
    },
    ...
  }
}
```

- `dates` are ISO-format; ascending order (oldest → newest). The port converts ascending to the computation form it needs (pct_change() is order-aware).
- Missing trading days are simply omitted (market holidays, etc.). Correlation pair-up uses the intersection of available dates per pair.
- `closes` are in native ccy; vol and correlation are scale-invariant so no INR/USD distinction is needed inside this skill.

### 2.3 `holdings.json` shape

```jsonc
{
  "as_of": "2026-04-17",
  "holdings": [
    { "ticker": "RELIANCE.NS", "market_value": 120000.0, "allocation_pct": 12.0,
      "sector": "Energy" },
    ...
  ]
}
```

- `market_value` and `allocation_pct` are both carried; the engine uses `allocation_pct` directly for HHI weights (converted to fractions: `w_i = alloc_pct_i / 100`). `market_value` is tolerated for producer-side diagnostics but not consumed by the math.
- `sector` is passed through into the output for downstream prose but not used in math.
- Derivable from `temp/research/portfolio-snapshot.csv` (which carries `allocation_pct` directly) — an M8 Phase 1 mapping concern.

### 2.4 Output JSON shape (SPEC §9.5 + investigation §6)

```jsonc
{
  "as_of": "2026-04-17",
  "lookback_days": 60,              // vol lookback (upstream default)
  "base_limit_pct": 8.0,            // from philosophy position_sizing.max_per_stock_pct
  "hhi": 0.1423,                    // Σ (alloc_pct_i / 100)²
  "holdings": [
    { "ticker": "RELIANCE.NS",
      "allocation_pct": 12.0,
      "annualized_volatility": 0.287,
      "vol_adjusted_limit_pct": 8.5,          // base_limit_pct × vol_multiplier × 100
      "delta_pct": 3.5 },                     // allocation_pct − vol_adjusted_limit_pct
    ...
  ],
  "top_correlation_pairs": [
    { "tickers": ["A.NS", "B.NS"], "correlation": 0.89 },
    ...
  ],
  "warnings": []                    // ["no price data for X", ...] for missing tickers
}
```

- `hhi` is rounded to 4 decimals for stable JSON output.
- Per-holding fields rounded to 3 decimals (vol) / 2 decimals (percentages).
- `top_correlation_pairs` sorted by `abs(correlation)` descending, then lexicographically by `tickers` tuple to break ties deterministically.
- `warnings` is the "holdings ticker with no matching prices entry" path — graceful: set `annualized_volatility: null`, `vol_adjusted_limit_pct: null`, `delta_pct: null`, exclude from correlation, emit a warning. Exit 0.

### 2.5 Determinism (investigation §5.3)

- No `random.*` (the test generator uses a seeded `numpy.random.RandomState`; the fixture JSON is committed; the engine itself never randomizes).
- Float ordering: `hhi` from explicit summation in holdings order; `holdings` emitted in input order; `top_correlation_pairs` sorted by `(−|ρ|, tickers)`.
- Rounding done at emit time via a helper so repeated runs produce byte-equal stdout.

### 2.6 Insufficient data

- A holdings ticker absent from `prices.json.tickers` → warning + null fields (see §2.4). Exit 0.
- A ticker with < 2 closes → same treatment (vol is undefined).
- Fewer than 2 tickers usable for correlation → `top_correlation_pairs: []`, no error.

## 6. Fixture design — generator + committed JSON

`tests/scoring-engine/gen_prices_fixture.py`:

- Seeded `np.random.RandomState(42)`.
- 10 tickers: `RELIANCE.NS`, `INFY.NS`, `HDFCBANK.NS`, `TCS.NS`, `BHARTIARTL.NS`, `ITC.NS`, `LT.NS`, `HINDUNILVR.NS`, `MARUTI.NS`, `COALINDIA.NS` (names only — numbers are synthetic).
- 252 trading days, monotone ISO dates back-dated from `2026-04-17`.
- Per-ticker GBM walk: `S_t = S_{t-1} * exp(μ_i dt + σ_i √dt × z_t)`, where some pairs share a common shock component to produce realistic correlation clusters (e.g. two IT tickers share ~60% of the daily shock).
- `holdings.json` — 10 tickers with `allocation_pct` summing to 100, deliberately skewing one ticker above `max_per_stock_pct` so the `delta_pct` signal is non-zero in at least one row.
- The generator is both committed AND run once at scaffold time; the committed `prices.json` / `holdings.json` are what tests load. Re-running the generator reproduces byte-identical files (determinism).
