## Approach

### Shim shape — three "plain" shims (canonical pattern, identical to M1.2)

`claim-scoring.sh`, `check-scoring-progress.sh`, `reclaim-scoring-stale.sh` each follow the 6–8 line pattern from [M1.2](M1.2-phase2-shims.md#approach):

```bash
#!/usr/bin/env bash
# <shim-name>.sh — Phase 4 <one-line behavior>. SPEC §10.5, §18.8.
# Usage: <shim-name>.sh <args-per-§18.8>
set -euo pipefail
# shellcheck source=./_claim-common.sh
source "$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)/_claim-common.sh"
CLAIM_ROOT="${REPO_ROOT}/temp/research/scoring-claims"
_claim_<entrypoint> "$@"     # or no "$@" for check-scoring-progress
```

### Shim shape — `complete-scoring.sh` (the only one that differs)

```bash
#!/usr/bin/env bash
# complete-scoring.sh — Phase 4 mark ticker(s) scoring-complete. SPEC §10.5, §18.8.
# Usage: complete-scoring.sh <ticker> [<ticker>...]
# Rejects (non-zero, no .done) if the stock file lacks a `## Scoring` header.
set -euo pipefail
# shellcheck source=./_claim-common.sh
source "$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)/_claim-common.sh"

_validate_scoring_header() {
  local stock_file="$1"
  grep -qE '^## Scoring[[:space:]]*$' "$stock_file"
}

CLAIM_ROOT="${REPO_ROOT}/temp/research/scoring-claims"
CLAIM_VALIDATOR=_validate_scoring_header
_claim_complete "$@"
```

Three things to note:

1. **Source first, then define the validator and set `CLAIM_ROOT`/`CLAIM_VALIDATOR`.** Same reason as M1.2: `_claim-common.sh` resolves `REPO_ROOT` from its own `${BASH_SOURCE[0]}` (or `$CLAIM_CTL_ROOT` test override) at [`_claim-common.sh:17`](../../skills/scripts/_claim-common.sh#L17). Sourcing first gives the shim a resolved `REPO_ROOT` to compose `CLAIM_ROOT` against. Defining the validator after `source` is fine because `_claim_complete` doesn't dereference `CLAIM_VALIDATOR` until it's called.
2. **Validator is a shell function name, not an inline command.** `_claim_complete` invokes `"$CLAIM_VALIDATOR" "$stock_file"` ([`_claim-common.sh:137`](../../skills/scripts/_claim-common.sh#L137)). Bash functions defined in the shim live in the same shell symbol table the library reads from — no special export needed.
3. **Header-only check** (investigation §6.5). The grep matches `^## Scoring` followed by optional trailing whitespace. Body validity (numbers, period annotations, threshold pass/fail) is out of scope — that's the stock linter (M9) and the Phase 3 verifier.

### Stock-file path convention

`_claim_complete` resolves the stock file to `${REPO_ROOT}/temp/research/stocks/${ticker}.md` ([`_claim-common.sh:120`](../../skills/scripts/_claim-common.sh#L120)). The shim doesn't need to re-derive this; the validator just receives the absolute path. If the stock file doesn't exist, `grep` exits non-zero with a "No such file" message on stderr — the library treats that as a validation failure (no `.done` written), which is the correct behavior (you can't complete scoring for a ticker that has no stock file).

### File mode

All four shims `chmod +x`. They're invoked directly (`skills/scripts/claim-scoring.sh 5 agent-A`) and via `bin/run-skill skills/scripts/claim-scoring.sh 5 agent-A` (Docker venue, investigation §1).

## Rejected alternatives

| Alternative | Why rejected |
|---|---|
| Put `_validate_scoring_header` in a separate sourced file (`skills/scripts/_scoring-validator.sh`) | The function is 2 lines and used by exactly one shim. A 5th file in `skills/scripts/` whose name doesn't appear in SPEC §18.8 dilutes the "one file per sub-command" contract finalized in investigation §3. Keep it inline. |
| Define the validator **before** sourcing the library | Works (Bash function defs are global), but inverts the M1.2 ordering of "source first, then configure" and means the validator references no library state. Trivial preference; M1.2's order wins for consistency across all shims. |
| Move the validator into `_claim-common.sh` itself, gated by a phase enum | Re-introduces the per-phase branching the library was designed to avoid (investigation §3, Option B). Validators are phase-specific; configuration belongs in the phase shim. |
| Validate the body (e.g. presence of every `[scoring-engine]` line, threshold tags) | Out of scope per investigation §6.5: header-only is the contract. Body validation is a stock-linter / Phase 3 verifier concern (M9). Coupling M1 to body content also blocks shipping the shim until those linters land. |
| Forward `"$@"` from `check-scoring-progress.sh` to `_claim_check_progress` | Same rationale as M1.2: SPEC §18.8 documents no args. Library accepts an optional minutes arg as a private knob; not part of the public CLI surface. |
| Validate `<agent-id>` format inside `claim-scoring.sh` | Investigation §7.5 leaves agent-id format to the caller. Validating here would diverge from the Phase 2 / Phase 5 shims. |

## Risks and open questions

- **R1. Validator function in same symbol table as library.** Bash function definitions are process-global; defining `_validate_scoring_header` in the shim and then having `_claim_complete` call `"$CLAIM_VALIDATOR" "$stock_file"` works because both run in the same shell process. If a future caller invokes `_claim_complete` from a subshell that hasn't sourced the shim, the function won't exist — but the shim *is* the entry point, so that path doesn't exist. Documented; not a real risk.
- **R2. Stock file may not exist when `complete-scoring` runs.** In a partial-failure scenario (Phase 2 produced no stock file but a Phase 4 caller tries to mark scoring complete), `grep` against a missing file exits non-zero with "No such file or directory" on stderr. `_claim_complete` treats this as a validation failure (no `.done` written, exit non-zero) — which is the correct behavior. No special handling needed in the shim.
- **R3. `## Scoring` regex tolerates trailing whitespace only, not trailing content.** `^## Scoring[[:space:]]*$` rejects `## Scoring (Q3 FY26)` if anyone tries to suffix the header. SPEC §10.5 doesn't authorize a suffix; investigation §6.5 explicitly chose the strict form. Document if a stock-file author trips on this; do not relax the regex without a SPEC change.
- **R4. `set -euo pipefail` from shim conflicts with library default.** Same no-op as M1.2 R1; both files set it independently for self-contained readability.
- **No open questions for the user.** The two design choices (validator placement: inline; validator scope: header-only) are decided in this plan and traceable to the investigation. Trivially reversible if Phase 3 verifier surfaces a need.
