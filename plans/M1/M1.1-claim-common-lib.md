## Approach

### Library shape

A pure-functions sourced file. Variables and functions all prefixed `_claim_` to avoid colliding with shim locals. Set-once configuration is read from shell variables set by the shim *before* `source`-ing:

| Variable | Required | Set by | Meaning |
|---|---|---|---|
| `CLAIM_ROOT` | yes | shim | Absolute path to one of `temp/research/claims/`, `temp/research/scoring-claims/`, `temp/research/persona-claims/`. Resolved relative to `REPO_ROOT` (see below). |
| `CLAIM_VALIDATOR` | no | shim | Name of a shell function `_validate_<x> <stock_file_path>` that returns 0/non-zero. Used only by `_claim_complete`. Default: empty (skip validation). |
| `CLAIM_MODEL_ID` | no | env | Optional model-ID string written into the `.claimed` / `.done` payload (investigation §6.7). |
| `CLAIM_CTL_ROOT` | no | env | Test-only override of `REPO_ROOT`. Lets fixtures point at an isolated temp tree (investigation §6.1). |

`REPO_ROOT` is derived once from `${CLAIM_CTL_ROOT:-$(cd "$(dirname "${BASH_SOURCE[0]}")/../.." && pwd)}` — script-relative resolution so callers can `cd` anywhere.

### Public API (functions called from shims)

| Function | Used by | Behavior |
|---|---|---|
| `_claim_batch <batch-size> <agent-id>` | `claim-stocks.sh`, `claim-scoring.sh`, `claim-persona.sh` | Lex-sort candidate ticker dirs under `$CLAIM_ROOT`, attempt atomic claim of up to N. One claimed ticker per stdout line. Exit 0 even on zero claims. |
| `_claim_complete <ticker> [<ticker>...]` | `complete-*.sh` | For each ticker: prerequisite-check `.claimed` exists; idempotency-check `.done`; run `$CLAIM_VALIDATOR` if set; write `.done`. Multi-arg partial-failure: continue past failures, exit non-zero if any ticker failed (investigation §7.2). |
| `_claim_check_progress` | `check-*-progress.sh` | One-line key/value summary: `total=N available=N claimed=N completed=N stale=N` (investigation §7.3). |
| `_claim_reclaim_stale [<minutes>]` | `reclaim-*-stale.sh` | Remove `.claimed` files older than threshold (default 30, configurable via CLI arg). Operates on `.claimed` only — never touches `.done` (investigation §7.1). |

### Internal helpers (not called by shims)

| Function | Purpose |
|---|---|
| `_claim_assert_root` | Guard: `CLAIM_ROOT` set + dir exists. Error to stderr + non-zero exit if not. |
| `_claim_one <ticker_dir> <agent-id>` | Single-ticker noclobber attempt. Returns 0 on win, 1 on loss. Caller iterates. |
| `_claim_payload <kind> <agent-id>` | Compose `.claimed` or `.done` payload string (key/value, investigation §6.7). |
| `_claim_mtime_now_minus <minutes>` | Cross-platform past-timestamp helper. macOS: `date -v-${m}M '+%Y%m%d%H%M.%S'`; GNU: `date -d "${m} min ago" '+%Y%m%d%H%M.%S'`. Used internally by `_claim_check_progress` for the stale count and by future tests (investigation §4). |

### The atomic claim primitive

Per investigation §1, the kernel-level guarantee is `O_CREAT|O_EXCL`. Bash exposes this through `set -o noclobber` plus the `>` redirect:

```bash
_claim_one() {
  local ticker_dir="$1" agent_id="$2"
  local claimed="${ticker_dir}/.claimed"
  # noclobber makes `>` fail (non-zero) if the target exists.
  # The subshell scopes the option so we don't leak it to the caller.
  if ( set -o noclobber; printf '%s\n' "$(_claim_payload claimed "$agent_id")" > "$claimed" ) 2>/dev/null; then
    return 0
  fi
  return 1
}
```

Two racing processes both try the redirect; the kernel guarantees exactly one wins. The losing branch is a clean exit-1 — no recovery needed because nothing was created.

### Availability rule

A ticker dir is available iff it contains **neither `.claimed` nor `.done`** (investigation §7.1). `_claim_batch` candidate scan:

```bash
# Lex-sort for reproducible races (investigation §6.2)
find "$CLAIM_ROOT" -mindepth 1 -maxdepth 1 -type d \
  ! -exec test -e '{}/.claimed' \; \
  ! -exec test -e '{}/.done' \; \
  -print | sort
```

Iterate, attempt `_claim_one` until `batch-size` wins or candidates exhausted. Print each winning ticker (basename of dir) on its own stdout line.

### Stale detection

`find` with `-mmin +<n>`:

```bash
find "$CLAIM_ROOT" -mindepth 2 -maxdepth 2 -type f -name '.claimed' -mmin "+${minutes}" -delete
```

`-mmin` is supported on both BSD `find` (macOS host) and GNU `find` (Debian container). `-delete` is available on both.

For `_claim_check_progress`'s `stale=` count (without deleting), same `find` minus `-delete`, piped to `wc -l`. Threshold for the count uses `${CLAIM_STALE_MINUTES_DEFAULT:-30}`.

### Completion-validation hook

The hook is a shell function name, not an inline command — keeps the shim's call site readable and lets shellcheck see the function body in the shim file:

```bash
# In skills/scripts/complete-scoring.sh (M1.3, illustrative)
_validate_scoring_header() {
  local stock_file="$1"
  grep -qE '^## Scoring[[:space:]]*$' "$stock_file"
}
CLAIM_ROOT="${REPO_ROOT}/temp/research/scoring-claims"
CLAIM_VALIDATOR=_validate_scoring_header
source "$(dirname "$0")/_claim-common.sh"
_claim_complete "$@"
```

Inside `_claim_complete`, the validator is invoked with the resolved stock file path `${REPO_ROOT}/temp/research/stocks/${ticker}.md`. The library does not own knowledge of *what* to validate — it only owns *when* to call the validator.

### Payload format (investigation §6.7)

```text
agent_id=agent-A
claimed_at_epoch=1766959200
model_id=gpt-5.2
```

`model_id` line emitted only if `CLAIM_MODEL_ID` is non-empty. Same shape for `.done`, with `completed_at_epoch` instead of `claimed_at_epoch`. Consumers must not parse — existence is the signal (investigation §7.1).

### Style requirements

- `set -euo pipefail` at the top of the library (and in shims).
- All variables quoted; `[[ ]]` instead of `[ ]`.
- `# shellcheck shell=bash` directive at top so shellcheck lints a sourced-only file correctly.
- All errors to stderr, exits non-zero with a usage hint.

## Rejected alternatives

| Alternative | Why rejected |
|---|---|
| One monolithic `claim-ctl.sh` script with a sub-command dispatcher (Option A from investigation §3) | Investigation finalized Option B (shared library + thin shims). Shims match SPEC §18.8's per-command file names directly and make `shellcheck skills/scripts/*.sh` a one-line gate. |
| Heartbeat thread instead of mtime stale detection | Investigation §2: mtime + 30-min budget is sufficient; heartbeat adds complexity (timer subprocess, bookkeeping) for no win on a single-host workflow. |
| `flock` instead of `O_EXCL` | `flock` is BSD/GNU-portable but adds a daemon-style `flock <fd> { … }` wrapping idiom that's awkward inside short-lived shims. `O_EXCL` via `noclobber` is the SPEC-mandated primitive (§6.7). |
| Validating `agent_id` ownership at `complete` time | Investigation §7.2 explicitly disallows this — recovery flows let a different session finish a claimed ticker. |

## Risks and open questions

- **R1. macOS `find -delete` + `-mmin +<n>` corner cases.** Mitigation: keep the find expression simple (one `-name '.claimed'`, one `-mmin`, one `-delete`); cover in M1.7 stale-reclaim test against both BSD and GNU find.
- **R2. `set -o noclobber` leaking to caller.** Mitigation: the noclobber attempt runs in a `( … )` subshell — investigation §6.7 atomic primitive snippet preserves this.
- **R3. Empty `.claimed` payload after SIGKILL between `>` and `printf`.** Mitigation: `printf` happens *before* `>` evaluates the redirect target, so the kernel atomically creates the file with the full payload — no partial write window. (`echo` would be similar but `printf` is portability-safer.) Documented in investigation §6.4: progress reporting must not parse `.claimed`, only count it.
- **R4. Test fixture isolation.** Mitigation: `CLAIM_CTL_ROOT` env var lets M1.6/M1.7 fixtures redirect to a `mktemp -d` tree without touching real `temp/research/`.
- **Open Q (for user):** Beads dependency direction on M1.1 is reversed (depends on M1.2/M1.3/M1.4 instead of being depended-on by them). Should I flip with `bd dep` calls before claiming the task, or leave as-is?
